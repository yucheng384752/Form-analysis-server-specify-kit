#  專案組織快速參考

## 檔案分類原則

###  **docs/** - 文檔資料夾
**用途**: 存放所有專案相關文檔
**包含**:
- 產品需求文檔 (PRD.md, PRD2.md)
- 專案說明 (README.md)
- 重構計劃 (RESTRUCTURE_PLAN.md)
- 啟動指南 (MANUAL_STARTUP_GUIDE.md)

### 🛠️ **scripts/** - 系統腳本
**用途**: 存放自動化腳本和系統管理工具
**包含**:
- 系統啟動/停止腳本
- 監控腳本
- 診斷工具

###  **test-data/** - 測試資料
**用途**: 存放各種測試用的 CSV 檔案
**結構**:
```
test-data/
├── P1/     # P1 類型檔案（生產批次資料）
├── P2/     # P2 類型檔案（原料資料）
├── P3/     # P3 類型檔案（檢驗資料）
└── *.csv   # 通用測試檔案
```

###  **legacy-components/** - 舊版組件
**用途**: 保存原始的 React 組件供參考
**包含**:
- 舊版 TSX 組件
- 已被替換的 UI 元件

###  **tools/** - 開發工具
**用途**: 開發和測試相關的 Python 腳本
**包含**:
- 驗證測試工具
- 匯入修復工具
- 快速啟動工具

###  **form-analysis-server/** - 主應用
**用途**: 完整的前後端應用程式
**結構**:
```
form-analysis-server/
├── backend/    # FastAPI 後端
├── frontend/   # React 前端
└── docker-compose.yml
```

## 🎯 使用指南

### 新增檔案時的分類規則

1. **文檔類** → `docs/`
   - .md, .pdf, .docx 等文檔檔案

2. **腳本類** → `scripts/`
   - .bat, .ps1, .sh 等系統腳本

3. **測試資料** → `test-data/`
   - .csv, .xlsx 等測試用資料檔案
   - 按 P1/P2/P3 分類

4. **工具程式** → `tools/`
   - .py 開發輔助工具

5. **舊版代碼** → `legacy-components/`
   - 不再使用但保留參考的程式碼

### 🚨 注意事項

- **不要** 將實際上傳的檔案提交到版本控制
- **保留** test-data 中的測試檔案
- **定期清理** uploads/ 資料夾
- **更新** README.md 當結構改變時
