# 常見錯誤（Troubleshooting）

> 更新日期：2026-01-20

本文件用來記錄在本專案（Form-analysis-server-specify-kit）啟動、建置與執行過程中，常見且可快速排除的錯誤。

---

## 1) `docker-compose` 建置前端時出現 BuildKit snapshot 不存在

### 現象
使用 `scripts\\start-system.bat` 啟動系統時，在建置前端（frontend）階段失敗，錯誤訊息類似：

```
[frontend] exporting to image:
------
target frontend: failed to solve: failed to prepare extraction snapshot "extract-... sha256:...":
parent snapshot sha256:... does not exist: not found
```

### 原因
Docker BuildKit / buildx 的快取（snapshot）資料損毀或不同步，導致建置過程在「匯出映像」或「解壓 layer」時找不到父 snapshot。

這通常是 Docker Desktop 長時間運作、異常關閉、或快取狀態不一致所引起，與專案程式碼本身無直接關係。

### 快速修復（建議先做）
在專案的 `form-analysis-server` 目錄下，對前端做一次 no-cache 重建：

```
docker-compose build --no-cache frontend
```

接著再啟動：

```
docker-compose up -d frontend
```

或直接重跑：

```
.\scripts\start-system.bat
```

### 進一步修復（若仍失敗）
1. 清理 BuildKit builder 快取：

```
docker builder prune -af
```

2. 重新啟動 Docker Desktop 後再試一次。

### 專案側的改善
`start-system.bat` 已加入自動重試：若 `docker-compose up -d --build frontend` 失敗，會自動改用 `docker-compose build --no-cache frontend` 重建後再嘗試啟動，以降低使用者手動清快取的頻率。
