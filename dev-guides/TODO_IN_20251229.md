### 1) PDF→v2 import 串接決策與落地（P0/P1 級別：路徑收斂）

- 決策（已確認）
  - PDF→CSV 的輸出先建立成 `UploadJob`（可檢視/手動修 CSV）。
  - 使用者手動觸發 commit；validate 可由 ingest 建立的 v2 import job 自動跑一次，若後續有編修 UploadJob 則可再重送 from-upload-job 建立新的 v2 job。
  - 最終寫入仍走 v2 import jobs 的 parse/validate/commit（UploadJob 只做可編修 staging）。
  - 規格文件：見 [dev-guides/PDF_TO_V2_IMPORT_SPEC.md](dev-guides/PDF_TO_V2_IMPORT_SPEC.md)

- API/行為變更（最小可落地版本）
  - 既有：`POST /api/upload/pdf/{process_id}/convert/ingest?skip_validate=true`
    - 行為：建立 UploadJob；`skip_validate=true` 時不做 upload validate（停在可編修階段）
    - 已補（2026-01-23）：此 endpoint 同時會為每個 UploadJob 建立 v2 `ImportJob`，並在回應內回 `import_job_id`（冪等：同 conversion job 重打會回同一批 ids）
  - 新增：`POST /api/v2/import/jobs/from-upload-job`
    - 產出：回傳 `ImportJobRead`（觸發 background parse+validate）
    - 行為：以 UploadJob.file_content（含使用者修正後內容）建立 v2 import job
  - 既有：`POST /api/v2/import/jobs/{job_id}/commit`

- 需修改檔案（候選）
  - `form-analysis-server/backend/app/api/routes_upload.py`（掛新 endpoint 或串接 flow）
  - `form-analysis-server/backend/app/services/pdf_conversion.py`（轉檔完成後 auto-ingest 需避免自動 validate；或改成只建立 UploadJob）
  - `form-analysis-server/backend/app/api/routes_import_v2.py` / `form-analysis-server/backend/app/services/import_v2.py`（新增 from-upload-job 建立 job 的能力）
